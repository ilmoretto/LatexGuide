Aula 1: Classes de Documento — A Fundação do LaTeX

Seja bem-vindo(a) ao Módulo 2!

No módulo anterior, aprendemos a linguagem básica da construção. Agora, vamos estudar as diferentes "plantas baixas" que definem o tipo de obra.

A escolha da \documentclass não é apenas uma formalidade; é a decisão arquitetônica mais impactante que você tomará, determinando o esqueleto, a escala e as capacidades do seu documento.

1. As Classes de Documento: Definindo a Natureza do Projeto

Vamos detalhar as três classes fundamentais, explorando suas características e usos ideais, conforme compilado a partir dos manuais técnicos e guias especializados.

Classe article

Pense na classe article como a planta de um apartamento: autocontido, eficiente e direto ao ponto.

Uso Ideal: Artigos científicos para periódicos, ensaios curtos, relatórios de laboratório, listas de exercícios, apresentações de conferência.

Hierarquia Máxima:

A principal unidade estrutural é a \section. Não há conceito de "capítulo". A hierarquia completa é:

  • \\section
  • \\subsection
  • \\subsubsection
  • \\paragraph
  • \\subparagraph
Layout Padrão:
  • Título: O \maketitle coloca o título no topo da primeira página, e o texto do documento começa logo abaixo. Não há uma página de rosto separada por padrão.
  • Impressão: Configurada para oneside (um lado), o que significa que as margens esquerda e direita são simétricas.
  • Numeração: Equações, figuras e tabelas são numeradas sequencialmente ao longo de todo o documento (e.g., Equação 1, Equação 2, ...).

Classe report

A classe report é como a planta de uma casa: múltiplos cômodos (seções) organizados em andares (capítulos).

Uso Ideal: Trabalhos de Conclusão de Curso (TCCs), dissertações de mestrado, teses de doutorado, relatórios técnicos extensos, documentação de software.

Hierarquia Máxima:

A principal unidade é o \chapter. A numeração dos elementos (seções, equações, figuras) é reiniciada a cada capítulo.

Layout Padrão:
  • Título: O \maketitle gera uma página de rosto separada.
  • Capítulos: Cada \chapter inicia automaticamente em uma nova página.
  • Impressão: Ainda configurada para oneside por padrão, mas frequentemente usada com a opção twoside.
  • Abstract: Possui um ambiente abstract que gera uma página de resumo separada.

Classe book

A classe book é a planta de um grande edifício ou de um condomínio, com múltiplas seções organizadas em blocos.

Uso Ideal: Livros, teses muito longas e complexas.

Hierarquia Máxima:

Adiciona um nível acima do capítulo, o \part, para agrupar grandes seções do livro.

Layout Padrão:
  • Impressão: Configurada por padrão para twoside, otimizada para impressão frente e verso. As margens internas (medianiz) são menores que as externas.
  • Capítulos: Não apenas começam em uma nova página, mas por padrão, em uma página ímpar (à direita), deixando uma página em branco se necessário.
  • Elementos Pré-textuais: Possui comandos específicos como \frontmatter, \mainmatter e \backmatter para controlar a numeração das páginas (romana para o prefácio, arábica para o conteúdo principal, etc.).

2. As Opções da Classe: Ajustando os Detalhes da Planta

As opções, passadas entre colchetes, são os "ajustes finos" do seu projeto. Vamos ver exemplos práticos para as mais importantes.

10pt, 11pt, 12pt

Exemplo: \documentclass[12pt]{article}

Efeito: Define a fonte base do corpo do texto para 12 pontos, tornando a leitura mais confortável em muitos casos. Títulos e outros elementos serão redimensionados proporcionalmente.

a4paper

Exemplo: \documentclass[a4paper]{report}

Efeito: Configura as dimensões da página para o padrão A4 (210mm x 297mm), o mais comum no Brasil e na Europa. Sem isso, o padrão é o letterpaper americano.

twoside

Exemplo: \documentclass[twoside]{report}

Efeito: Informa ao LaTeX para formatar o documento para impressão frente e verso. O resultado mais visível são as margens assimétricas (a margem interna é menor) e os cabeçalhos que podem ser diferentes para páginas pares e ímpares.

twocolumn

Exemplo: \documentclass[twocolumn]{article}

Efeito: Formata todo o corpo do documento em um layout de duas colunas, comum em artigos científicos e newsletters. O LaTeX gerencia o fluxo de texto entre as colunas automaticamente.

landscape

Exemplo: \documentclass[landscape]{article}

Efeito: Inverte as dimensões de altura e largura da página, criando um layout em modo paisagem. Ideal para apresentar tabelas muito largas ou figuras panorâmicas.

titlepage

Exemplo: \documentclass[titlepage]{article}

Efeito: Força a classe article (que normalmente não o faz) a criar uma página de título separada para as informações do \maketitle. O texto principal começará na página seguinte.

3. Exemplo Prático Combinado

Vamos criar um pequeno documento usando a classe report e algumas opções para ver o efeito combinado. Este exemplo demonstra como a escolha da classe e das opções define fundamentalmente o resultado.

Código-Fonte (main.tex)

% Classe 'report', fonte 12pt, papel A4, layout frente e verso.
\\documentclass[12pt, a4paper, twoside]{report}

\\usepackage[utf8]{inputenc}
\\usepackage[T1]{fontenc}
\\usepackage[brazil]{babel}

\\title{Relatório Abrangente Sobre a Tipografia Digital}
\\author{Equipe LaTeX}
\\date{Setembro de 2025}

\\begin{document}

% A opção 'titlepage' é padrão no 'report', então o título terá sua própria página.
\\maketitle

% O ambiente 'abstract' também gera uma página separada no 'report'.
\\begin{abstract}
    Este é o resumo do nosso relatório, apresentando os principais pontos 
    que serão discutidos nos capítulos a seguir.
\\end{abstract}

\\tableofcontents

\\chapter{A Era do TeX}
Como a classe é 'report', este capítulo iniciará uma nova página. A numeração das seções
será vinculada a este capítulo.

\\section{O Problema da Qualidade}
Donald Knuth estava insatisfeito com a qualidade tipográfica da época.

\\chapter{A Chegada do LaTeX}
% Como a opção 'twoside' está ativa, o LaTeX pode deixar a página 4 em branco
% para garantir que este capítulo comece em uma página ímpar (página 5).
Este segundo capítulo também iniciará em uma nova página.

\\section{Simplificando o Processo}
Leslie Lamport criou macros para facilitar o uso do TeX.

\\end{document}

O que observar ao compilar este código:

  1. Página 1 (Folha de Rosto): O título estará sozinho em uma página.
  2. Página 2 (Resumo): O conteúdo do ambiente abstract terá sua própria página.
  3. Página 3 (Sumário): O sumário será gerado aqui.
  4. Página 4 (Pode estar em branco): Se a opção openright (padrão em book e às vezes em report com twoside) estiver ativa, esta página pode ser deixada em branco para que o próximo capítulo comece na página 5 (uma página da direita).
  5. Página 5 (Capítulo 1): O primeiro capítulo começa aqui.
  6. Página 7 (Capítulo 2): O segundo capítulo também começa em uma nova página (e também em uma página ímpar).
  7. Margens: Ao visualizar o PDF, você notará que as margens não são iguais. Nas páginas ímpares, a margem esquerda será maior; nas páginas pares, a direita será maior, preparando o documento para encadernação.

4. Atividade Prática (Seja o Arquiteto - 20 minutos)

Missão: Experimentar com Classes e Opções

  1. Use o código do Exemplo Prático Combinado acima.
  2. Experimento 1: Mude a classe de report para book. Recompile. Quais diferenças sutis você nota?
    (Dica: olhe a numeração das páginas pré-textuais e o texto nos cabeçalhos, se houver)
  3. Experimento 2: Volte a classe para report. Remova a opção twoside. Recompile. O que aconteceu com as margens das páginas?
  4. Experimento 3: Mantenha a classe report e a opção twoside. Adicione a opção twocolumn. Recompile. Observe como o LaTeX refaz todo o layout do seu relatório complexo para um formato de duas colunas, mantendo a estrutura de capítulos e seções.

Com este conhecimento detalhado, você agora pode:

  • Tomar decisões informadas sobre a melhor estrutura para qualquer tipo de documento
  • Compreender como as opções de classe afetam o layout final
  • Escolher between article, report e book com base nas necessidades específicas
  • Aplicar opções como twoside, twocolumn e titlepage estrategicamente
